# Project Documentation

## Structure

| Folder | Purpose |
|--------|---------|
| [`ai/`](./ai/) | Navigation and orientation for AI assistants |
| [`standards/`](./standards/) | Project standards: conventions, testing rules, DTO examples |
| [`architecture/`](./architecture/) | Architecture decisions, DDD layers, AI agent design |
| [`ideas/`](./ideas/) | Space for product ideas in projects created from this starter |
| [`tasks/`](./tasks/) | Space for backlog and current tasks in derived projects |
| [`plans/`](./plans/) | Development plans, roadmaps, sprints |
| [`api/`](./api/) | API documentation (endpoints, contracts, OpenAPI) |

## Key files

- `ai/start.md` — **start here** (AI navigation, mandatory reading list)
- `standards/conventions.md` — coding conventions (language, PHP, TypeScript, naming, git)
- `standards/project-context.md` — product overview and key domains
- `standards/testing-standards.md` — testing rules for backend, frontend, and Playwright
- `standards/dto-examples.md` — DTO patterns with plain readonly objects
- `architecture/ddd.md` — DDD layers, patterns, anti-patterns, architectural tests
- `architecture/frontend.md` — frontend structure, module lifecycle, testability rules
- `architecture/ai-agent.md` — AI agent design (LLM routing, budget control, data collection)
- `architecture/decisions.md` — ADR log: key architectural decisions and their rationale
- `ideas/README.md` — how to capture and structure product ideas
- `tasks/README.md` — task tracking conventions for derived projects
